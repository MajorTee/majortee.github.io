C学习（二）

发布于 2019-10-03 评论 编程语言  C语言 

指针

指针释放问题（野指针）

char* p = NULL;

//注意要先给空间才能cpy值
p = (char*)malloc(100);

strcpy(p, "111111");
if(p != NULL)
{
    free(p);
}
// 不小心进行两次free
if(p != NULL)
{
    free(p);
}

虽然第一次 p 指向的地址 0xaa11 区域的东西 free 了，但是 p 本身还是指向 0xaa11 ，此时一段垃圾区域（野指针），再free就出现问题了

因此需要

if(p != NULL)
{
    free(p);
    p = NULL;
}

指针或函数共享内存块

char* p1 = NULL;
char* p2 = NULL;
p2 = (char*)malloc(100);
int i;
for(i = 0; i < 10; i++)
{
    p1 = p2 + i;
}

此时 p1，p2 共享同一内存空间，可以类比两个函数也共享同一内存空间

void main()
{
    int a = 0;
    int* p = NULL;
    p = &a;
    other(p);
}

void other(int* e)
{
    *e= 2;
}

二级指针

char* p1 = NULL;
char** p2 = NULL;
//直接修改
p1 = 0x11;
p2 = 0x22;
//间接修改
p2 = &p1;
*p2 = 0xaa; // -> p1 = 0xaa

跟共享内存结合

void getMem1(char** p)
{
    *p = 0xcc;
}
void getMem2(char* p)
{
    p = 0xcc;//修改无效,只是形参，不是对地址值操作
}
getMem1(&p1);//对p1的地址值位置内存进行操作
getMem1(p1);//修改无效

写个例子

#pragma warning(disable:4996)
void main()
{
   char* p1 = NULL;
    int len1 = 0;
    char* p2 = NULL;
    int len2 = 0;

    getMem(&p1, &len1, &p2, &len2);
    printf("len1: %d, len2: %d \n", len1, len2);// len1: 7, len2: 7
    printf("len1: %s, len2: %s \n", p1, p2);// len1: 1234565, len2: abcdefg
    printf("len1: %c, len2: %c \n", *p1, *p2);// len1: 1, len2: a

    if(p1 != NULL)
    {
        free(p1);
        p1 = NULL;
    }
    if(p2 != NULL)
    {
        free(p2);
        p2 = NULL;
    }
}

void getMem(char** myp1, int* mylen1, char** myp2, int* mylen2)
{
    char* temp1, * temp2;

    temp1 = (char*)malloc(100);
    strcpy(temp1, "1234565");
    temp2 = (char*)malloc(201);
    strcpy(temp2, "abcdefg");

    //间接赋值
    *myp1 = temp1; //一级指针间接赋值
    *myp2 = temp2;
    *mylen1 = strlen(temp1);//二级指针间接赋值
    *mylen2 = strlen(temp2);

}

字符串

字符数组

// 指定长度
char buff[100] = {'a', 'b', 'c'};//其它值为0
// 不指定
char buff[] = {'a', 'b', 'c'};//是个数组，不是字符串，字符串以0结尾

字符串初始化字符数组

char buff[] = "abcdef";// 作为字符数组长度为7个字节，作为字符串是6个
// 求字符长度
int len = strlen(buff);// 6
// 数组所占内存空间大小
int size = sizeof(buff); // 7

[] 和 *

buff[i] ===> buff[0+i] ===> *(buff+i)

buff 是一个常量指针，析构内存时，保证buff所指向的内存空间安全释放

指针易错点

char 越界

char buff[3] = "abc";//还要考虑 \0

优先级问题

++比*优先级高，因此注意*p++的问题
(*p)++  ->p指向的数据自加
*p++    ->p地址值自加

注意二级指针自加问题

//两种方式
char** buffa = buff1;
char* buffb = *buff2;

//*((*buff1copy)++) = *(source + i);
**buffa = *(source + i);
(*buffa)++;

*buffb++ = *(source + i);

第一种方式行不通，虽然因为buffa拷贝buff1，同时建立A->B->C，但是当B改变了，那么buffa和buff1所指向的内存位置同时改变，因此拷贝没有意义

第二种方式有效，等于使用一个D = B->C，其中B改变了，使C一直增长，但原始的A->B->C还在指向C的首地址

const 问题

const char* c; //c不可以修改
char* const d;// 类似char buff[],首地址不可以修改，char*不可修改
const char* const e;

c 是一个指向常量整形数的指针，所指向的内存空间不可以修改，本身地址值可以修改

d 是一个常量指针，它所指向的内存空间可以修改，本身地址值不可以修改

e 是一个指向常量整形数的常量指针,指向的内存空间以及本身均不可以修改

绕过 const 变量

const int a = 10;
int* p = &a;
*p = 100;
printf("%d", a);// -> 100
