二级指针

对空间的释放

void main()
{
    char* p = NULL;
    get_mem(&p)
}

void get_mem(char** p)
{
    *p = (char*)malloc(100);
}
void free_mem(char** p)
{
    if(p != NULL)
    {
        free(*p); //注意不是**p
        *p = NULL;
    }
}

反例

void free_mem1(char* p) //传递一级指针
{
    if(p != NULL)
    {
        free(*p); //注意不是**p
        *p = NULL;
    }
}

void free_mem2(char* vovo) //传递一级指针
{
    if(vovo != NULL)
    {
        free(vovo); //注意不是**p
        vovo = NULL;
    }
}

在此函数内对p = NULL，只是把一个名字一样的形参置为NULL，对实参无影响，如 free_mem2，造成野指针问题

三级指针初始化易错点

当使用一个二级指针拷贝三级指针中二级指针地址，使用拷贝的二级指针malloc一块空间，跟三级指针中二级指针地址关联性出现问题

char*** str = NULL；
char** tmpStr = NULL;
*str = (char**)malloc(100);//是对的
tmpStr = *str;
printf("*str: %p, tmpStr:%p", *str, tmpStr);
//-> *str: 011E60F8, tmpStr:011E60F8


char*** str = NULL；
char** tmpStr = NULL;
tmpStr = *str; //由于*str初始化他们都指向了0
tmpStr = (char**)malloc(100);//此时tmpStr指向了0xaa11,str还是0
printf("*str: %p, tmpStr:%p", *str, tmpStr);
//-> *str: CCCCCCCC, tmpStr:00E860F8
// 可以在*str = tmpStr;来解决

对它的一级指针也是一样

char* realTmpStr = NULL;
*tmpStr = (char*)malloc(100);
realTmpStr = *tmpStr;

问题原因开始未初始化的指针都指向0，初始化后指向一个新地址，而原指针依旧指向0。

如A->B->0->0，tmp = B，tmp->0xaa->0，此时原指针依旧 A->B->0->0，->寻址路线在创建时就固定了，只改变中间的地址值，->寻址路线未改变，因此出现问题

先使用二级指针操作，最后将三级指针指向二级指针最好的方法
