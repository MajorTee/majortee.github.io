tenserflow 运行方式

计算图：

tenser – n维的矩阵

计算图的产生 – 使用tenseroard使计算可视化

eg: 

    import terserflow as tf 
    a = tf.add(2, 3)

输出：操作符，变量，常量

session 运行tenser并获得值

sess = tf.Session()
print sess.run(a)
sess.close()

github 有sess.run() 函数代码，注释清晰

不用close的方式
# with clause takes care of sess.close()
with tf.Session() as sess:
    print sess.run(a)
其他计算图
x = 2
y = 3
op1 = tf.add(x, y)
op2 = tf.multiply(x, y)
op3 = tf.pow(op2, op1)
with tf.Session() as sess:
    op3 = sess.run(op3)


Mul已经弃用

首先在Pow结点，依次根据需要去找 Multiply 结点和 Add 结点
tenser不会预先计算Multiply和Add 之所以这样是因为并非所有节点都需要计算，节省

更多结点图
x = 2
y = 3
op1 = tf.add(x, y)
op2 = tf.multiply(x, y)
useless = tf.multiply(x, op1)
op3 = tf.pow(op2, op1)
with tf.Session() as sess:
    op3 = sess.run(op3)

反向依赖传播

session把要计算的点送到fetches list

同时可以指定某个节点到某些GPU，CPU运行（分布）

当运行多个计算图时，每个都有自己的session，都需要获取资源，每个图的换金相互独立，不能传递数据或者信息，除非到Numpy层

tf.Graph()

sess = tf.Session(graph=g) 中的参数即表明这个session在这个Graph中运算

g = tf.get_default_graph()
可以得到默认图环境

不要混用默认图和自己定义的图
